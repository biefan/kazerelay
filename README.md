## ✨ 核心特性

### 🎯 智能推流
- **双模式架构**：云端中继 / 本地直推，自由切换
- **一键启动**：自动完成从登录到推流的全流程
- **智能重连**：网络波动自动恢复，无需人工干预

### 🔒 安全隔离
- **账号隔离**：每个账号独立会话，互不干扰
- **代理支持**：SOCKS5 / HTTP 代理，支持认证
- **多区域节点**：智能选择最优服务器，降低延迟

### 🚀 性能优化
- **低延迟传输**：优化的推流协议，延迟 < 2秒
- **断线托管**：云端自动守护，断线秒级恢复
- **资源占用低**：单账号 < 100MB 内存

### 💼 企业级管理
- **多账号管理**：支持批量操作，统一监控
- **授权系统**：安全的许可证管理机制
- **日志追踪**：完整的操作日志，便于排查

## 🎬 快速开始

### 1. 添加账号
点击 **"添加账号"** → 浏览器自动打开 → 完成登录 → 自动导入

### 2. 获取推流权限
选择账号 → 点击 **"获取推流码"** → 选择推流模式 → 完成

### 3. 开始推流
- **方式一（推荐）**：点击 **"一键推流"** → OBS 自动配置并开始推流
- **方式二**：手动复制推流地址到 OBS → 手动启动

## 🌟 高级功能

### 云端中继模式
```
本地 OBS → 云端中继服务器 → 目标平台
```
**优势**：
- ✅ 稳定性高，云端自动重连
- ✅ 支持代理推流，IP 隔离
- ✅ 断线托管，无人值守

### 本地直推模式
```
本地 OBS → 本地中继 → 目标平台
```
**优势**：
- ✅ 延迟更低
- ✅ 节省流量成本
- ✅ 完全可控

## 💡 使用建议

### 适合云端中继的场景
- 需要长时间稳定推流
- 网络环境不稳定
- 需要使用代理服务
- 多账号批量管理

### 适合本地直推的场景
- 对延迟要求极高
- 网络环境优秀
- 短时间推流
- 节省流量成本

## ⚙️ 配置说明

### OBS 设置（一键推流时自动配置）
- **视频编码器**：x264 或 NVENC（推荐）
- **码率**：2500-4000 kbps
- **关键帧间隔**：2 秒
- **CPU 使用预设**：veryfast 或 faster

### 网络优化
- 建议使用有线网络
- 上传带宽建议 > 5 Mbps
- 如使用代理，确保代理稳定性

## ❓ 常见问题

<details>
<summary><b>Q: 推流地址获取失败怎么办？</b></summary>

A: 
1. 检查网络连接是否正常
2. 确认账号是否有推流权限
3. 尝试切换服务器节点
4. 查看详细错误日志
</details>

<details>
<summary><b>Q: 云端托管是什么？</b></summary>

A: 云端托管功能可以让服务器在检测到推流断开时自动重连，无需人工干预。适合长时间无人值守的场景。
</details>

<details>
<summary><b>Q: 如何选择推流模式？</b></summary>

A: 
- 如果需要稳定性和自动恢复，选择 **云端中继**
- 如果追求极致延迟，选择 **本地直推**
</details>

<details>
<summary><b>Q: 为什么推荐使用代理？</b></summary>

A: 使用代理可以实现 IP 隔离，避免多账号关联风险，提升账号安全性。
</details>
